# AIVibe — UX, удержание и конверсия

> Рабочий документ для владельца / команды.
> Разбор пользовательского сценария (journey) «скан → AI-расстановка → примерка → замена → покупка»:
> что реально построено в коде, что ещё нет, какие фичи стоит заимствовать у лидеров, что психологически удерживает
> и как повысить желание покупать и оформлять подписку.
> Статусы реализуемости — **по факту из кода** (а не по описанию в документах).

## 1. Резюме на одной странице

Сценарий — **правильное видение**, совпадающее с тем, что делают IKEA Kreativ и Wayfair. Но проверка по коду показала **большой разрыв между сценарием и тем, что реально построено**.

- **AR-ядро (расстановка по эргономике) — реально и сделано хорошо.** Это ваш главный технический актив.
- **«AR-first без LiDAR» — кнопка-заглушка.** Де-факто приложение работает **только на iPhone Pro** с LiDAR → отсекается массовый рынок РФ.
- **Шаг 4 («заменить на похожий») и Шаг 5 (промокод/оплата/CPA) в коде НЕ существуют** — это новые модули, а не доработки.
- ~~**Бэкенд всё ещё ищет товары скраперами WB/Ozon** (Apify) — противоречит модели «партнёрства без scraping».~~ ✅ Решено 2026-06-10: Apify-путь удалён, единственный источник — партнёрский каталог YDB.
- **Главный пробел удержания:** после одной комнаты **нет повода вернуться**.

**5 приоритетов роста** (детально — §7): путь без LiDAR · каталог фабричных USDZ · детерминированный движок как основной путь · шаг 4 (замена) + пере-чек коллизий · решить «партнёры vs Apify» и контур CPA.

## 2. Реализуемость по шагам (проверка по коду)

| Шаг сценария | Статус | Что в реальности |
|---|---|---|
| 1. Скан 30 сек (LiDAR) | ✅ Реально | Честный RoomPlan, прогресс, авто-стоп. ОК для iPhone Pro |
| 1b. Ввод размеров без LiDAR | ❌ Заглушка | Кнопка есть, но действие — пустышка (`case .manualEntryTapped` → `.none`). Экрана ввода нет → **только Pro-устройства** |
| 2. «3–4 сек» генерация | ⚠️ Риск | Синхронный LLM (таймаут 25 сек, без стриминга), ИИ сам считает координаты + ретрай. Реально 5–15 сек |
| 3. Расстановка по эргономике | ✅ Реально (сильно) | `GenerateArrangementTool` + `CollisionService` + `DesignNorms` — зрелый детерминированный движок |
| 3b. Фотореалистичные USDZ | ⚠️ Частично | Загрузчик (`USDZLoader`) есть, но **каталога моделей фабрик нет** → сейчас покажет placeholder-геометрию |
| 4. «4 похожих по габаритам/стилю» | ❌ Нет в коде | Действия замены не существует (`ARDesignerFeature` умеет tap/remove/refine, но не «alternatives»). Новая фича |
| 5. Промокод + deeplink + вебхук CPA | ❌ Нет в коде | Весь монетизационный контур — только в документах |
| Бэкенд: «нативный вектор YDB» | ⚠️ Риск | На деле `scan('rag_chunks', limit 500)` + косинус в Node на Document API — **не** ANN/KNN |
| Бэкенд: источник товаров | ✅ Решено (2026-06-10) | Apify-скраперы WB/Ozon удалены, источник — партнёрский каталог YDB |

> **Как сделать «3–4 сек» честными:** пусть ИИ выбирает только *список мебели и стиль*, а *координаты* считает детерминированный движок (миллисекунды) + стримить текст. Тогда «материализация» в AR — 1–2 сек.

### 2.1. Регуляторные мины, которые сценарий пропускает

- **App Store (anti-steering, Guideline 3.1.1):** deeplink из приложения на оплату на стороннем сайте — риск отклонения. Оплату уводить аккуратно, не «в лоб» из iOS-UI.
- **152-ФЗ:** если в вебхук / на сайт фабрики уходит идентификатор пользователя — это передача ПДн третьим лицам, нужно согласие + отражение в политике.
- **ЗоЗПП / реклама:** «30 сек», «±10–15 см», «3–4 сек» — проверяемые обещания об услуге, не просто маркетинг.
- **Защищённое ядро:** переключение основного пути расстановки на детерминированный движок затрагивает `ToolRegistry`/`AgentLoop` (по CLAUDE.md — только bugfix/добавление) → нужно согласование владельца.
- **Замена предмета** обязана **заново прогонять коллизии** (даже +10% по габаритам может задеть дверь) — дёшево, движок уже есть.

## 3. За и против сценария

**За:** эмоционально верная драматургия (скан → магия → примерка ногами → замена → покупка); совпадает с лидерами рынка; AR-ядро реально; no-deps бэкенд дисциплинирован; промокод-CPA продуман в документах.

**Против:** половина «вау» не построена (шаги 4–5, путь без LiDAR, каталог USDZ); latency приукрашен; код тянет старую scraping-модель; нет петли возврата; шаг 5 несёт юр-риски (Apple / 152-ФЗ).

## 4. Инновации из известных приложений (проверено в вебе)

| Источник | Фича | Как встроить в AIVibe |
|---|---|---|
| **IKEA Kreativ** | «Scene Scanner» + **цифровое удаление старой мебели**; «дизайн в рамках бюджета» | Дать «очистить комнату» (показать пустое пространство) — важно для не-новых квартир. Режим «уложиться в бюджет» |
| **Wayfair AI** | Загрузил **фото** → выбрал стиль → ИИ заполнил → **«купить всё сразу»** | Вход **по фото** (без скана) как низкобарьерная альтернатива; кнопка **«Купить весь интерьер»** (↑ средний чек) |
| **Houzz** | «View in My Room 3D» + **Ideabooks** (доски/коллекции) | Сохранённые проекты и доски вдохновения → повод вернуться, шеринг |
| **Pinterest** | «Shop the Look» — тап по предмету на картинке → купить | Тап по любому предмету в AR → карточка + альтернативы (это и есть ваш шаг 4) |
| **Duolingo / Spotify** | Стрики, цели, годовой «Wrapped», before/after | Re-engagement: «обновите интерьер к осени», сезонные тренды, «было/стало» для шеринга |
| **BNPL РФ (Подели / Яндекс Сплит / Долями)** | Рассрочка повышает конверсию **до +50%** и средний чек **×2–5** | Показывать «от X₽/мес» на карточке — для мебели (чек 25–45к) это сильнейший рычаг покупки |

> Бонус-аргумент для маркетинга и для фабрик: AR-примерка, по отраслевым оценкам, заметно снижает возвраты мебели — это и продающий тезис, и реальная польза партнёрам.

## 5. Психология удержания: где сильно, где слабо

| Механизм | В сценарии | Как усилить |
|---|---|---|
| **IKEA-эффект / владение** | ✅ Силён: пользователь «сам обставил» свою комнату → ценит | Сохранять «Мою комнату», дать имя проекту, «ваш дизайн» — усилить чувство собственности |
| **Эффект Зейгарник (незавершённость)** | ⚠️ Слабо | Прогресс: «комната готова на 70% — добавьте освещение» тянет закончить |
| **Вариативная награда** | ✅ «Магия расстановки» | Кнопка «ещё вариант» — дозированный дофамин, как лента соцсети |
| **Пик-энд** | ⚠️ Частично | Момент материализации в AR сделать кинематографичным (пик); финал — шеринг «было/стало» (хороший «энд») |
| **Соц. доказательство** | ❌ Нет | «N человек поставили этот диван», тренды недели (этично, без фейков) |
| **Петля крючка (возврат)** | ❌ **Главный пробел** | После одной комнаты нет причины вернуться. Нужны: новые поступления партнёров, сезонные тренды, мультикомната, пуши «новый диван подходит к вашей гостиной» |

**Вывод:** сценарий силён в моменте «здесь и сейчас» (владение + магия), но **проваливает удержание во времени**. Это приоритет №1 для роста LTV.

## 6. Playbook конверсии: покупать и подписываться

### 6.1. Покупать мебель

1. **Снизить трение шага 5.** Ручной ввод промокода + редирект — самый «протекающий» и рискованный (Apple) шаг. Целевое: **оплата внутри / сплит ЮKassa** → «Купить» в 1–2 тапа, атрибуция 100%.
2. **BNPL на карточке** (Подели / Сплит) — для мебели это +50% конверсии и ×2–5 чек. Сильнейший быстрый рычаг.
3. **«Купить весь интерьер»** (корзина из всех предметов) — поднимает средний чек, как у Wayfair.
4. **Ловить эмоцию сразу:** в момент материализации в AR показывать «купить всё / сохранить / поделиться», пока эмоция горячая (пик-энд).
5. **Доверие:** реальные цены, фото-отзывы, «в наличии», сроки доставки, возврат (AR и так снижает возвраты).

### 6.2. Оформлять подписку PRO (599₽)

1. **Что закрыть в PRO:** безлимит сканов/вариантов, **HD-рендер, экспорт PDF/USDZ, «замена на похожий» (шаг 4 — самая залипательная фича), мультикомната**; в FREE — водяной знак на шеринге.
2. **Когда показывать пейволл:** после первого «вау», при экспорте/шеринге, на 2-й комнате, при попытке «заменить на похожий».
3. **Якорение цены:** FREE / PRO 599 / BUSINESS 2490 — BUSINESS делает PRO «дешёвым» (эффект приманки). Годовая со скидкой (4990/год).
4. **Дизайнеры (главные плательщики PRO):** позиционировать как «автоматизируй рутину, выигрывай клиентов AR-обходом».
5. ⚠️ **Apple:** подписку продавать **на сайте/лендинге**, в приложении показывать ценность, но не вести прямую ссылку на внешнюю оплату цифровой подписки (anti-steering).

## 7. Что строить первым (приоритеты)

1. **Путь без LiDAR (ручной ввод размеров)** — без него нет массового рынка РФ. Сейчас разрыв №1.
2. **Каталог фабричных USDZ + резолвинг по артикулу** (а не из ответа ИИ) — иначе шаг 3 = заглушки.
3. **Детерминированный движок как основной путь расстановки** (скорость + стабильность; согласовать — это ядро).
4. **Шаг 4 «заменить на похожий»** — атрибутный поиск по каталогу (категория + габариты ±10–15% + стиль) + **обязательный пере-чек коллизий**.
5. **Решить «партнёры vs Apify»** и контур CPA (это один блок: каталог + промокод + вебхук).
6. **Петля возврата** (повод вернуться) + **BNPL** — рычаги LTV и конверсии.

## 8. Методология и источники

- Статусы реализуемости — из кода: `AIVibe/Features/RoomScan/RoomScanFlowFeature.swift` (`.manualEntryTapped` → `.none`), `AIVibe/Features/ARDesigner/` (`GenerateArrangementTool`, `CollisionService`, `DesignNorms`, `USDZLoader`, `ARDesignerFeature`), `backend/functions/ai-advisor/index.js` (синхронный LLM, таймаут 25 сек), `backend/shared/rag-search.js` (full-scan + косинус), `backend/functions/marketplace/index.js` (Apify-скраперы).
- Фичи конкурентов — проверены веб-поиском: IKEA Kreativ (Scene Scanner, удаление мебели), Houzz «View in My Room 3D» + Ideabooks, Wayfair AI / Room Planner, Pinterest «Shop the Look».

- Психология — стандартные модели: IKEA-эффект, эффект Зейгарник, вариативное подкрепление, Hook Model, пик-энд правило.
- Канон цен подписки — `STRATEGY.md` §1.3 (FREE / PRO 599₽ / BUSINESS 2490₽). Связанный документ — `docs/BUSINESS_MODEL.md`.
- Регуляторика — App Store Guideline 3.1.1 (anti-steering), 152-ФЗ (передача ПДн третьим лицам), ЗоЗПП (достоверность обещаний об услуге).
